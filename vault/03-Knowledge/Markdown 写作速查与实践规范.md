# Markdown 写作速查与实践规范

## 一句话定义
- Markdown 是轻量级标记语言，核心价值是“纯文本可读 + 跨平台可维护”。

## 适用场景
- 日报、项目记录、知识沉淀。
- 需要长期版本管理（Git）与跨设备编辑的文档。

## 核心要点
- 标题层级保持清晰：`#` 到 `###` 足够覆盖大多数场景。
- 优先使用列表与代码块组织信息，减少大段连续文本。
- 链接优先双链（`[[...]]`）表达知识关系，标签用于分类。
- 代码块必须标注语言，如 ` ```cpp ` / ` ```bash `。

## 常见误区
- 误区 1：把 Markdown 当成排版工具，过度使用 HTML 样式。
- 误区 2：只打标签不建双链，导致图谱关系稀疏。
- 误区 3：文件命名随意，后续检索和跨平台同步困难。
